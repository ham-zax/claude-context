Satori Architecture

Satori separates control plane and compute plane. The MCP runtime handles tool contracts, state, sync orchestration, and access gates. The core engine handles chunking, embedding, vector search, and delta reindexing. On Zilliz Cloud (Milvus) free tier, Satori can index up to 5 codebases (one collection per codebase).

6 tools. Runtime-first retrieval. Call-graph sidecar. Snapshot gates. Merkle sync. MCP-compatible for any agent.

Runtime Boundaries

MCP Runtime (@zokizuan/satori-mcp)

  • Defines the 6-tool MCP contract.
  • Validates tool inputs via Zod-derived JSON Schema.
  • Enforces snapshot fingerprint compatibility.
  • Schedules background sync and optional watcher sync.
  • Applies capability-based defaults (including rerank behavior).

Core Engine (@zokizuan/satori-core)

  • Parses files and splits code at structural boundaries.
  • Generates embeddings and performs dense/hybrid retrieval.
  • Maintains deterministic chunk IDs and line metadata.
  • Computes Merkle-diff file deltas for incremental reindex.
  • Stays MCP-agnostic and reusable outside MCP.

Tool Surface

Satori intentionally exposes only six MCP tools to reduce routing ambiguity while keeping call-chain traversal and file navigation first-class.

Tool Purpose Key Params Notes
manage_index Create/sync/status/clear index lifecycle. action, path, optional force, splitter Primary state transition entrypoint.
search_codebase Runtime-first semantic search with grouped/raw output. query, scope, resultMode, groupBy, limit, optional debug Grouped mode emits callGraphHint for direct graph jumps.
call_graph Traverse callers/callees from a symbolRef. symbolRef, direction, depth, limit Uses prebuilt sidecar index; deterministic node/edge order.
file_outline Sidecar-backed symbol outline for one file. path, file, start_line, end_line, limitSymbols Returns per-symbol callGraphHint handles and hasMore truncation signal.
read_file Safe file reads with optional line ranges and annotated mode. path, start_line, end_line, mode Annotated mode returns content + outlineStatus without failing content reads.
list_codebases List tracked roots and state. None Fast readiness and inventory check.

State and Safety

Snapshot state controls searchable access. Compatibility mismatches are blocked until explicit reindex.

Fingerprint Fields

{ provider, model, dimension, vectorStore, schemaVersion }

Safety Result

No silent serving of stale or incompatible embeddings.

Search And Sync Behavior

Search Pipeline 
query -> pass1 + expanded pass2 retrieval
  -> RRF fusion
  -> scope hard filter (runtime/docs/mixed)
  -> path priors (entrypoint/core/docs/tests)
  -> grouped/raw shaping
  -> grouped result includes callGraphHint + freshnessDecision
Ignore Model 
1) Built-in defaults
2) Constructor overrides
3) Env custom values
4) Repo ignore files (.gitignore, .satoriignore)
5) Global ignore (~/.satori/.satoriignore)

Runtime-first defaults reduce docs/test noise without requiring aggressive query-time excludes.

Capability Resolver + Rerank 
capability profile (fast/standard/slow)
  -> default candidate limits
  -> rerank behavior decided internally
  -> no public useReranker toggle on search_codebase

Default limit ranges vary by profile (fast/standard/slow) and provider locality.

Incremental Sync 
hash files -> diff Merkle DAG -> {added, removed, modified}
removed/modified -> delete old chunks
added/modified   -> split + embed + insert

Polling loop runs every 3 minutes, optional watcher reuses the same delta pipeline.
